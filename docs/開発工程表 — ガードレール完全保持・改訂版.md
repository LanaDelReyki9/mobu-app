# 開発工程表 — ガードレール完全保持・改訂版

【ガードレールの目的】
・アセット作りに凝りすぎて燃え尽きるのを防ぐ
・HTML / CSS / JS という「アプリの骨格」を先に完成させる
・完成度より「最後まで到達すること」を最優先する

==================================================

## Phase 0：準備と最小アセット確定（例外的に画像あり）

目安時間：4〜6時間

目的：
・再現性が失われる前に「顔と体型の基準」を固める
・ただし“作り込みは禁止”

- [x]  0-1 VS Code + index.html / style.css / script.js 作成
- [x]  0-2 CSS変数定義（色・フォント）
- [x]  0-3 【マスター確定】　モブ君 Ver.4 バストアップ（中立・無光源・透過）　※ 最高品質だが1枚のみ。ここで止める。
- [x]  0-4 【差分耐性チェック】　Ver.4 → Ver.1 / Ver.2 / Ver.3 が　同一人物として生成できるかを「確認」するだけ　※ 完成させない。出ればOK。
- [x]  0-5 基本アイコンSVG 5種（最低限）

【Phase 0 ガードレール】
□ Ver.1〜3を完成させようとしていないか？
□ 背景・光・演出を考えていないか？
→ 1つでもYESなら即中断してPhase1へ
